parsing the command line, validates input integrity and fills the arg structure. CAUTION: This function allocates memory which must be freed. The command line expected to be of the form: 0. program name.

  1. [MUST]input file name string: holdes the full/relative path for the text file to be cyphered. MUST
  2. [MUST]encryption key: the key the program will use to encrypt or decrypt the file.
  3. [OPT] number of threads: the number of threads to use to encrypt or decrypt the file. Default: 1
  4. [OPT] encrypt/decrypt flag: indication weather to encrypt or decrypt the file, the option are '-e' and '-d'. Defualt: encrypt.
    Parameters
    [in]argc_
    [in]argv_
    [in]p_output_args
    Returns
    error_e 0 (ERROR_OK) on success, nonzero on failure.
  1. [MUST]input file name string: holdes the full/relative path for the text file to be cyphered. MUST
  2. [MUST]encryption key: the key the program will use to encrypt or decrypt the file.
  3. [OPT] number of threads: the number of threads to use to encrypt or decrypt the file. Default: 1
  4. [OPT] encrypt/decrypt flag: indication weather to encrypt or decrypt the file, the option are '-e' and '-d'. Defualt: encrypt. Implemntation: Stright forward, reading the input, and constantly checking for the validity fo the input. Pitfalls: converting the string to int, allocating memory for strings.
